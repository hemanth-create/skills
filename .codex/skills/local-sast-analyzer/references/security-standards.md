# Security Standards Reference

Read this reference only when standards context materially improves a finding or report.

- [NIST Secure Software Development Framework](https://csrc.nist.gov/projects/ssdf): use it as a risk-based, adaptable secure-development reference, not as a fixed checklist.
- [OASIS SARIF 2.1.0](https://www.oasis-open.org/standard/sarif-v2-1-0/): use SARIF's structured result fields when normalizing an exported static-analysis report.
- [OWASP Code Review Guide](https://owasp.org/www-project-code-review-guide/): use it to frame code-review coverage and secure controls.
- [MITRE CWE](https://cwe.mitre.org/): map a finding to a weakness category only when the code evidence supports the mapping.

These references do not turn a pattern match or scanner result into a confirmed vulnerability. Local evidence, reachability, and controls determine the final classification.
